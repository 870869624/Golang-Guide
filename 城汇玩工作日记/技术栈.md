### 数据库迁移

- 迁移脚本：`migratedb/migrations/000001` ~ `000065`
- 每个版本含 `.up.sql` 和 `.down.sql`
- GORM AutoMigrate 映射 100+ 数据模型
- 60+ 张业务表：用户、订单、活动、部落、支付、分账、数字藏品、Web3钱包、商品、评价等

### Redis使用场景
- 基础KV缓存（`SetX/GetX/DelX`）
- 分页数据缓存（TTL=1分钟）
- Redis Stream消息流（消费者组模式，`XReadGroup`阻塞1分钟）

## 消息队列

| 技术 | 版本 | 用途 |
|------|------|------|
| streadway/amqp | v1.1.0 | RabbitMQ AMQP客户端 |
| nsqio/go-nsq | v1.1.0 | NSQ（历史遗留，已注释） |

### RabbitMQ 配置
- Exchange类型：**Direct**（持久化）
- ACK模式：**手动ACK**（`autoAck=false`）
- QoS：`prefetchCount=1`（一次处理一条，保证顺序）
- 失败处理：`Nack(false, true)` 重入队
- 成功处理：`Ack(false)` 确认

### 队列清单

| 队列名 | 消费内容 | 对应消费者 |
|--------|---------|-----------|
| `custingNft` | NFT铸造消息 | `createTokenId.go` |
| `shareOrder` | 分账订单消息 | `createTokenId.go` |
| `userCashout` | 用户提现消息 | `cashOut.go` |
| `businessCashout` | 商户提现/进件消息 | `cashOut.go` |
| `notice` | 通知消息 | Redis Stream消费者 |

### 消费模式
- `SubscribeToQueue()` — 直接订阅队列
- `SubscribeToExchange()` — Exchange + RoutingKey + Queue 绑定

## 支付系统

| 技术 | 版本 | 用途 |
|------|------|------|
| wechatpay-go | v0.2.18 | 微信支付V3官方Go SDK |
| shopspring/decimal | v1.3.1 | 高精度金额计算 |
| go-pay/gopay | v1.5.100 | 支付聚合库（备用） |
| tjfoc/gmsm | v1.4.1 | 国密算法（SM2/SM3/SM4） |

### 微信支付双实例
- 主商户客户端（`wechatPayClient`）：自营商品支付
- 合作伙伴商户客户端（`wechatPartnerPayClient`）：服务商模式，为子商户提供支付能力

### 调用的微信API
| API | SDK方法 | 用途 |
|-----|---------|------|
| JSAPI下单 | `jsapi.PrepayWithRequestPayment` | 小程序支付 |
| 退款 | `refunddomestic.RefundsApiService.Create` | 全额/部分退款 |
| 标准分账 | `ProfitSharing.OrdersApiService` | 分账 |
| 完结分账 | `OrdersApiService.UnfreezeOrder` | 解冻剩余资金 |
| 回退分账 | `ReturnOrdersApiService` | 分账回退 |
| 电商平台分账 | `/v3/ecommerce/profitsharing/orders` | 平台级分账 |

### 签名与证书
- `option.WithWechatPayAutoAuthCipher` 自动签名验签
- 私钥从PEM文件加载（`utils.LoadPrivateKeyWithPath`）
- 敏感字段用 `EM_APIV3` 加密（RSAES-OAEP）
- 证书目录：`cert/wxpay/`（主商户）、`cert/partnerwxpay/`（伙伴商户）

### 订单号生成
```
商户号前4位 + 时间戳hex + 4字节随机hex
```
使用 `crypto/rand` 保证密码学安全。
