## 二、程序启动流程

```go
// cmd/main.go
func main() {
    // 1. Bootstrap初始化（通过 import _ "uefun_api/bootstrap" 触发init()）
    //    Config → Log → Redis → DB → RabbitMQ → DB.Check → Model.Init → Migrate → AutoMigrate
    
    // 2. 创建context（支持优雅退出）
    ctx, cancel := context.WithCancel(context.Background())
    wg := sync.WaitGroup{}
    
    // 3. 启动6个并行goroutine
    wg.Add(6)
    go func() { defer wg.Done(); routers.Run(ctx) }()           // HTTP服务
    go func() { defer wg.Done(); nft.ConsumeMessagesCreateTokenId() }()  // NFT铸造
    go func() { defer wg.Done(); nft.ConsumeMessagesShareOrder() }()     // 分账
    go func() { defer wg.Done(); cashout.StartCashOut(ctx) }()           // 用户提现
    go func() { defer wg.Done(); cashout.StartBusinessCashOut(ctx) }()   // 商户提现
    go func() { defer wg.Done(); notice.ConsumeNoticeMessages(ctx) }()   // 通知
    
    // 4. 启动定时任务调度器
    systemTask.StartScheduler(ctx)
    
    // 5. Prometheus指标采集（每10秒）
    go collectMetrics(ctx)
    
    // 6. 优雅退出
    sigCh := make(chan os.Signal, 1)
    signal.Notify(sigCh, syscall.SIGINT, syscall.SIGTERM)
    <-sigCh
    
    cancel()   // 通知所有goroutine退出
    wg.Wait()  // 等待所有goroutine完成
}
```

## 三、优雅退出机制

```
SIGINT / SIGTERM
    │
    ▼
cancel() → 所有监听 context.Done() 的goroutine收到退出信号
    │
    ├── HTTP服务 → server.Shutdown(ctx) 优雅关闭（等待处理中的请求完成）
    ├── MQ消费者 → for range msgs 循环退出（channel关闭）
    ├── 定时任务 → scheduler.Stop() 停止所有cron job
    └── Prometheus采集 → 采集循环退出
    │
    ▼
wg.Wait() → 所有goroutine完成
    │
    ▼
进程退出
```

**关键点**：
- `context.WithCancel` 是Go服务优雅退出的标准模式
- `sync.WaitGroup` 确保所有goroutine退出后才结束进程
- MQ消费者通过 `for msg := range msgs` 自动在channel关闭时退出

## 四、依赖初始化链

```go
// bootstrap/init.go
func init() {
    // 1. 解析配置文件
    flag.StringVar(&configPath, "c", "config/1.0develop/config.toml", "config path")
    flag.Parse()
    viper.SetConfigFile(configPath)
    viper.ReadInConfig()
    
    // 2. 初始化日志（zap + lumberjack滚动切割）
    log.InitLogger()
    
    // 3. 初始化Redis
    redis.InitRedis()
    
    // 4. 初始化MySQL（GORM）
    db.InitDB()
    
    // 5. 初始化RabbitMQ
    mq.InitMQ()
    
    // 6. 数据库连接检查
    db.Check()
    
    // 7. 数据模型初始化
    model.Init(db.GetDB())
    
    // 8. 数据库迁移（golang-migrate）
    migratedb.Migrate()
    
    // 9. GORM AutoMigrate（可选，100+ 表）
    if config.Db.IsInit {
        db.GetDB().AutoMigrate(...)
    }
}
```

**`import _` 触发init()**：Go的 `import _ "package"` 会执行该包的 `init()` 函数但不导入任何符号。这是Go服务初始化的常见模式。

## 五、数据库版本管理

### 双重迁移机制

| 机制 | 工具 | 用途 |
|------|------|------|
| golang-migrate | `migratedb/migrations/` | 版本化SQL迁移（65个版本） |
| GORM AutoMigrate | `bootstrap/init.go` | 基于结构体自动同步表结构（100+表） |

### 迁移脚本结构

```
migratedb/migrations/
    000001_create_users.up.sql
    000001_create_users.down.sql
    000002_create_orders.up.sql
    000002_create_orders.down.sql
    ...
    000065_xxx.up.sql
    000065_xxx.down.sql
```

### 使用方式

```go
// migratedb/migrate.go
m, _ := migrate.NewWithSourceInstance(
    "file",
    source.NewFileSource("file://./migratedb/migrations"),
    "mysql://user:pass@tcp(host:port)/dbname",
)
m.Up()    // 执行所有未执行的up迁移
m.Down()  // 回滚最后一个迁移
```

## 六、监控体系

### Prometheus 指标

```go
// 每10秒采集一次
go func() {
    ticker := time.NewTicker(10 * time.Second)
    for {
        select {
        case <-ticker.C:
            var m runtime.MemStats
            runtime.ReadMemStats(&m)
            metrics.SetHeapAlloc(float64(m.HeapAlloc))
            metrics.SetHeapSys(float64(m.HeapSys))
            metrics.SetGoroutines(float64(runtime.NumGoroutine()))
        case <-ctx.Done():
            return
        }
    }
}()
```

### 采集的指标

| 指标 | 说明 |
|------|------|
| `uefun_heap_alloc` | 堆内存分配量 |
| `uefun_heap_sys` | 堆内存系统申请量 |
| `uefun_heap_idle` | 空闲堆内存 |
| `uefun_heap_inuse` | 使用中堆内存 |
| `uefun_goroutines` | Goroutine数量 |
| HTTP请求指标 | 请求数、延迟、状态码（Prometheus中间件自动采集） |

## 九、面试常见问题

### Q1: 6个goroutine同时启动，顺序怎么保证？
> 不需要保证顺序。它们是独立的消费者/服务，各自处理自己的任务。通过context.Done()统一协调退出。

### Q2: ReadMemStats会不会影响性能？
> 会。ReadMemStats会STW（Stop-The-World）。每10秒一次影响不大，但如果频率更高就需要考虑替代方案（如 `runtime/debug.ReadGCStats`）。

### Q3: 为什么同时用golang-migrate和GORM AutoMigrate？
> golang-migrate管理版本化的schema变更（可回滚），GORM AutoMigrate做快速表结构同步。生产环境应只用golang-migrate。

### Q4: 如果RabbitMQ连不上，服务还能启动吗？
> 当前设计下不能（`log.Fatal`会终止进程）。改进方案：延迟连接，或使用健康检查重试机制。
