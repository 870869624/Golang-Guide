### 模块概述

负责EDR产品的威胁情报管理，包括黑名单/白名单（哈希、路径）的CRUD、分组管理、批量导入导出、启用禁用等。

| 文件 | 功能 |
|------|------|

| `grouping_delete_logic.go` | 删除情报分组 |

| `info_import_template_download_logic.go` | 下载导入模板 |

### 1.1 情报分组管理

#### 创建分组（grouping_create）

```
业务流程：
1. 校验分组名称不为空
2. 查询同一租户下是否有同名同类型的分组（防重复）
3. 创建分组记录：grouping_name, is_used=false, tenant_id, type, created_at, updated_at
```

#### 分组的作用

- 情报可以属于多个分组（grouping字段是ID数组）
- 分组用于策略下发：导入情报后，异步通知Agent同步对应分组的策略
- 分组类型：blacklist（黑名单）、whitelisthash（白名单-哈希）、whitelistpath（白名单-路径）

### 1.2 情报导入（核心功能）

#### 业务流程

```
1. 接收Excel文件上传（multipart form）
2. 用excelize库读取Sheet1
3. 解析Excel表头作为字段key
4. 逐行解析：
   - 根据类型不同，解析不同字段：
     - 黑名单/白名单哈希：规则(rule)、类型(data_type)、风险名称、风险类型、状态、分组
     - 白名单路径：文件路径(file_path)、状态、分组
5. 加载已有数据，用于去重（防止重复导入）
   - 去重规则：
     - 黑名单/白名单哈希：rule + data_type
     - 白名单路径：file_path
6. 批量插入MongoDB（InsertMany）
7. 异步更新策略：
   - 收集导入数据涉及的所有分组ID
   - 检查分组是否需要更新策略
   - 调用Agent的PolicyRulesSync接口下发策略
```

#### 关键设计

##### Excel解析的健壮性
- 第一行作为表头
- 每行解析时检查index越界
- 分组名称自动解析为分组ID（查MongoDB匹配grouping_name）

##### 去重机制
```go
func (l *InfoImportLogic) getDedupKey(info *intelligence.Info, infoType string) string {
    if infoType == "blacklist" || infoType == "whitelisthash" {
        return info.Rule + "|" + info.DataType
    } else if infoType == "whitelistpath" {
        return info.FilePath
    }
    return ""
}
```
- 先加载全量已有数据到map
- 每解析一行检查去重key是否已存在
- 已存在则跳过，同时标记到map中防止本批次内重复

##### 哈希类型校验
```go
func (l *InfoImportLogic) isValidHashType(dataType string) bool {
    return dataType == "SHA256" || dataType == "SHA1" || dataType == "MD5"
}
```
只接受这三种哈希类型，其他忽略。

##### 异步策略同步
导入完成后，go协程异步调用Agent策略同步接口：
```go
go func() {
    _ = l.handleUpdatePolicy(req.Type, parsedData, tenantID)
}()
```
不阻塞导入接口的返回，提升用户体验。

### 1.3 情报导出

#### 业务流程

```
1. 根据筛选条件查询MongoDB
2. 格式化数据：
   - 时间戳转可读格式（2006-01-02 15:04:05）
   - enable布尔值转"启用"/"禁用"文本
   - 分组ID转分组名称（查MongoDB）
3. 生成Excel文件：
   - 根据情报类型选择不同的表头
   - 每100000行一个Sheet（支持大数据量导出）
4. 返回文件路径和文件名
```

#### 关键设计

##### 大数据量导出分Sheet
```go
var batchSize = 100000
sheetCount := totalRecords / batchSize
```
避免单个Sheet数据量过大导致Excel打开缓慢。

### 1.4 i18n国际化

所有错误提示使用i18n：
```go
message := i18n.T(l.ctx, "grouping name already exists")
```
支持多语言错误提示，不同语言环境返回不同语言的错误信息。

### 2.1 威胁查询 - 三种维度

#### 按终端分类查询（type=agent）

```
业务流程：
1. RBAC权限过滤：GetChildGroupIDsByRole获取用户管理的终端分组
2. 如果有权限限制，注入group_id $in条件
3. 支持筛选：principal_name, group_name, agent_ip, hostname（模糊匹配）
4. 分页查询ThreatAgentSummaryModel
5. 收集所有agentID，批量调用资产服务SearchAgent获取终端元信息
   - 操作系统类型（os_type）
   - 终端在线状态（agent_online）
   - 负责人（principal_name）
6. 填充元信息后返回
```

#### 按威胁分类查询（type=threat）

```
业务流程：
1. RBAC权限过滤（同上）
2. 支持筛选：virus_name, agent_ip, 时间范围
3. 分页查询ThreatVirusSummaryModel
4. 收集agentID，批量调用资产服务获取终端元信息
5. 构建ThreatVirusDetail列表返回
```

#### 溯源追踪查询（type=trace）

```
业务流程：
1. 按MD5查询时：
   - 使用TraceThreatNameFindByFilter（专用溯源查询，可能有特殊索引优化）
   - 返回全部匹配记录（不分页）
2. 按病毒名查询时：
   - 普通分页查询
3. 将log_type代码转中文：
   - edr-scan → 病毒查杀
   - edr-filemon → 文件实时监控
   - edr-behavior → 恶意行为监控
   - edr-udiskmon → U盘保护
   - edr-dlmon → 下载保护
   - edr-mail → 邮件监控
   - edr-threat → 高级威胁
```

### 2.2 RBAC权限过滤

```go
groupIDs := audit.GetChildGroupIDsByRole(l.svcCtx.RolesvcRPC, l.svcCtx.Agent, tenantID, l.GetRoleID())

if len(groupIDs) != 0 {
    filter["group_id"] = bson.M{
        "$in": stringGroupIDs,
    }

```

- 调用RolesvcRPC的GetChildGroupIDsByRole，根据角色递归获取管理的终端分组
- 如果返回空数组，说明用户有全部权限（不注入过滤条件）
- 如果返回非空数组，注入group_id过滤

### 2.3 威胁处理 - 单条/批量 + 同MD5处理

#### 业务流程

```
1. 校验参数：actionType和ids不能为空
2. 将ids转为ObjectID，查询MongoDB获取威胁记录
3. 根据actionType映射任务类型：
   - trust → threat_response_trusted（信任文件）
   - untrust → threat_response_trusted_cancel（取消信任）
   - restore → threat_response_recover_isolation（恢复隔离区）
   - clear → threat_response_update（病毒文件处理）
4. 如果req.SameHash=true：
   - 收集所有记录的MD5
   - 查询这些MD5对应的所有威胁记录（不仅限于选中的）
   - 按AgentID分组，每组发送一个批量任务
5. 如果req.SameHash=false：
   - 按AgentID分组，每组发送一个批量任务
6. 调用Agent的TaskSendNotCreateTask接口下发任务
```

#### 同MD5处理（SameHash）

这是一个重要功能：用户选中几条威胁记录处理后，可以选择"处理所有相同MD5的记录"。

```go
func updateSameThreat(updateInfo SameUpdateInfo) error {
    // 1. 收集所有去重MD5
    md5Set := make(map[string]bool)
    for _, info := range updateInfo.Infos {
        if !md5Set[info.MD5] {
            md5Set[info.MD5] = true
            md5s = append(md5s, info.MD5)
        }
    }
    // 2. 查询这些MD5对应的所有威胁记录
    filter := bson.M{"md5": bson.M{"$in": md5s}}
    allInfos, _ := updateInfo.svcCtx.Model.ThreatResponseModel.FindByFilter(...)
    // 3. 按AgentID分组下发任务
    agentGroups := make(map[string][]threatresponse.ThreatResponse)
    for _, info := range allInfos {
        agentGroups[info.AgentID] = append(agentGroups[info.AgentID], info)
    }
    for agentID, infos := range agentGroups {
        sendSameTaskToAgent(...)
    }
}
```

### 2.5 样本提取

#### 业务流程

```
1. 接收threat_response_ids列表
2. 转为ObjectID，查询MongoDB获取威胁记录
3. 过滤已删除/已清除的记录（handle == "deleted" || handle == "cleared"）
4. 按AgentID分组构建提取参数
5. 对每个Agent发送提取任务：
   - taskType: threat_response_extract
   - 参数包含：file_path, virus_path, md5, file_name, report_type, attach_id
6. 异步插入提取记录到ThreatFileModel（状态：提取中）
```

### 2.6 任务下发机制

所有操作最终都通过Agent的TaskSendNotCreateTask接口下发：

```go
_, err = l.svcCtx.Agent.TaskSendNotCreateTask(l.ctx, &pb.TaskSendNotCreateTaskReq{
    TenantId: tenantID,
    TaskType: "threat_response_extract",  // 或其他任务类型
    ExecType: "immediately",              // 立即执行
    Label:    "assets",                   // 资产标签
    Data: []*pb.TaskSendNotCreateTaskData{
        {
            Flag:   "device",
            FlagId: agentID,
            Param:  taskParams,  // structpb.Struct
        },
    },
})
```

- TaskSendNotCreateTask：下发任务但不创建任务记录（由调用方管理状态）
- ExecType: immediately：立即执行
- Label: assets：标记为资产管理类任务
- Data中的Param用structpb.NewStruct构建，支持任意结构化数据

### 3.1 高级威胁列表查询

#### 业务流程

```
1. 校验tenant_id
2. 构建查询条件：
   - tenant_id（必须）
   - result（处理结果，可选）
   - log_level（日志级别，可选）
   - threat_events（威胁事件，模糊匹配）
   - agent_ip（Agent IP，模糊匹配）
   - 时间范围（behavior_start_time）
3. 查询总数和分页数据
4. 计算命中技术数（hitTechCount）：
   - 优先使用Statistics.HitTechCount字段
   - 如果为空，遍历AttackMatrix累加Techs长度
5. 返回摘要列表
```

#### 关键设计

##### 正则安全转义
```go
filter["threat.threat_events"] = bson.M{"$regex": regexp.QuoteMeta(req.ThreatEvent), "$options": "$i"}
```
使用regexp.QuoteMeta转义用户输入，防止正则注入。

##### 分页防御
```go
skip := req.Skip
if skip < 0 {
    skip = 0
}
limit := req.Limit
if limit <= 0 {
    limit = 20
}
```

##### 手写parseInt
```go
func parseInt(s string) int {
    if s == "" {
        return 0
    }
    var result int
    for _, c := range s {
        if c >= '0' && c <= '9' {
            result = result*10 + int(c-'0')
        } else {
            break
        }
    }
    return result
}
```
手动解析数字字符串，避免引入strconv的额外开销（虽然实际差异不大）。

## 四、跨模块设计模式

### 4.1 统一的Agent任务下发

三个模块都使用相同的任务下发机制：
```go
l.svcCtx.Agent.TaskSendNotCreateTask(ctx, &pb.TaskSendNotCreateTaskReq{...})
```

- NGDLP：sensitive_file_extract
- NGEDR威胁响应：threat_response_extract, threat_response_trusted, threat_response_update等
- NGEDR情报：PolicyRulesSync

### 4.2 统一的RBAC权限过滤

威胁响应模块使用RolesvcRPC获取权限，注入查询过滤条件。

### 4.3 统一的错误处理

- 使用i18n.T进行错误消息国际化
- 使用logx记录结构化日志
- 使用regexp.QuoteMeta防止正则注入
