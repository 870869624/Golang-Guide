```bash

```

## 4. 手动触发合并 (这是一个异步操作，执行后会加入合并队列)

```bash
kubectl exec -it -n topsec-topngdlp statefulset.apps/clickhouse -- clickhouse-client --password TopsecCdu_1130 --query "OPTIMIZE TABLE sensitive_file FINAL;
"
```

### 4.1 可能会遇到

```bash
[root@master ~]# kubectl exec -it -n topsec-topngdlp statefulset.apps/clickhouse -- clickhouse-client --password TopsecCdu_1130 --query "OPTIMIZE TABLE sensitive_file FINAL;
> "
Received exception from server (version 22.8.8):
Code: 214. DB::Exception: Received from localhost:9000. DB::ErrnoException. DB::ErrnoException: Could not calculate available disk space (statvfs), errno: 2, strerror: No such file or directory. (CANNOT_STATVFS)
(query: OPTIMIZE TABLE sensitive_file FINAL;
)
command terminated with exit code 214
```

#### 4.1.1 错误原因分析

- ClickHouse 在执行 OPTIMIZE 等数据合并操作之前，需要检查目标表的存储路径是否有足够磁盘空间。它通过 statvfs 系统调用来获取路径所在文件系统的信息。
- errno 2 表示“No such file or directory”，说明传递给 statvfs 的路径在文件系统中找不到。
- 在 Kubernetes 环境中，最可能的原因是：

  - ClickHouse 的数据目录（默认 /var/lib/clickhouse）或临时目录（tmp_path）挂载的持久卷（PVC）未正确挂载。
  - Pod 启动后，持久卷的挂载点丢失（例如容器重启后 PVC 未能重新挂载，或挂载配置错误）。
  - ClickHouse 配置文件中指定的某个磁盘（如 storage_configuration 中的自定义路径）不存在。
